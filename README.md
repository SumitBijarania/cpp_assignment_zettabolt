# TPC-H Query 5 Implementation in C++ (Multithreaded)

This project implements TPC-H Query 5 ("Local Supplier Volume") in C++ using multithreading and raw `.tbl` data files generated by the TPC-H dbgen tool.

## Project Features

*   **Fully in-memory processing** (no DBMS)
*   **CLI-based parameter control** (region, date range, thread count, paths)
*   **Multithreaded order processing** for better performance
*   **Outputs sorted revenue per nation**

## Dependencies

*   C++17 or later
*   Standard C++ STL libraries (no external dependencies)
*   CMake 3.10+

## Steps to Run the Project

### 1. Clone or Download

```bash
git clone https://github.com/SumitBijarania/cpp_assignment_zettabolt.git
cd cpp_assignment_zettabolt
```

### 2. Generate TPC-H Data (Optional if you already have .tbl files)

Download and compile dbgen tool from the TPC-H benchmark.

1.  Go to the official TPC-H GitHub repository- https://github.com/electrum/tpch-dbgen
2.  Clone the repository:
    ```bash
    git clone https://github.com/electrum/tpch-dbgen.git
    cd tpch-dbgen
    ```
3.  The DBGEN tool is written in C, so you need to compile it.
    ```bash
    make
    ```
    *Note: If you encounter build errors on modern GCC, you may need to use flags like `-fcommon`.*

4.  Generate data and move to project directory:
    ```bash
    ./dbgen -s 1 -f                  # Generate SF1 (1GB) data
    mkdir -p ../data                 # Create data directory
    mv *.tbl ../data/                # Move files
    ```

### 3. Compile the Project

We use CMake for building:

```bash
mkdir -p build && cd build
cmake ..
make
```

### 4. Run the Executable

```bash
./tpch_query5 --r_name ASIA --start_date 1995-01-01 --end_date 1996-01-01 --threads 4 --table_path ../data/ --result_path ../output.txt
```

### 5. CLI Parameters:

*   `--r_name <REGION>`        : Region name to filter (e.g., ASIA)
*   `--start_date <DATE>`      : Start of order date (inclusive)
*   `--end_date <DATE>`        : End of order date (exclusive)
*   `--threads <N>`            : Number of threads for processing
*   `--table_path <PATH>`      : Path to directory containing .tbl files
*   `--result_path <FILE>`     : Output file path for final result

## How It Works - Step-by-Steptt

### The TPC-H Schema
[Tpch Schema](https://drive.google.com/file/d/1gfbhCIOMXQrj8jspw2VKzz3jtV6xXPWw/view?usp=sharing)

### Step 1: Data Preparation
Use `dbgen` to generate `.tbl` files for tables: region, nation, customer, supplier, orders, lineitem.

### Step 2: Read Data
Custom parsers read each `.tbl` file and convert rows into maps for efficient O(1) lookups. We optimize by loading only the necessary columns (Projections).

### Step 3: CLI Interface
User provides region, date range, thread count, paths. Parameters are parsed dynamically.

### Step 4: Multithreaded Processing
The `lineitem` table (the largest table) is divided among N threads.

Each thread:
1.  Iterates through its chunk of lineitems.
2.  Joins with Orders (filtered by date).
3.  Joins with Customer (filtered by Region/Nation).
4.  Joins with Supplier (filtered by Nation).
5.  Calculates local revenue if `CustomerNation == SupplierNation`.

### Step 5: Aggregate & Output
We use a **Map-Reduce** pattern. Each thread maintains its own local results. The main thread aggregates these local results, sorts them, and writes to `output.txt`.

## How Multithreading Boosts Performance

Here's the rationale behind the observed speedup when leveraging multithreading:

1.  **Parallel Workload Distribution**: Tasks are split across multiple threads, allowing simultaneous execution and significantly reducing total processing time.
2.  **Task Efficiency**: Multithreading accelerates CPU-bound tasks such as joining and aggregating data.

### Scalability with Threads
Performance typically improves with more threads—up to the point where thread management overhead (e.g., context switching) outweighs the benefits.
